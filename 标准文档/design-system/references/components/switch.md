# 开关（Switch）

## Overview

Switch 开关为基本组件之一；有两个互斥的选项（例如开/关、是/否、启动/禁用），用来打开或关闭某一选项的控件。选择其中一个选项会立即执行操作。

**设计师：** 陈亮  
**设计来源帧：** `Switch开关-规范`

---

## 使用原则

1. **文案不加逻辑词**：Switch 控件本身已能表现当前开/关状态，相关文案只需表示所控制的内容，不要在 List 或表单中写"打开 / 关闭"等词语
2. **操作后立即执行**：点击开关后立刻生效，无需额外确认步骤
3. **危险操作须二次确认**：若操作结果较为危险，点击开关后应加入二次确认流程，避免严重损失

---

## 状态

共四种状态，on / off 各自独立成组：

| 状态 | Figma 组件名 | 说明 |
|---|---|---|
| 打开 · 常规 | `开关/01on/01常规` | 默认可交互，已开启 |
| 打开 · 禁用 | `开关/01on/02禁用` | 已开启但不可操作，整体透明度 30% |
| 打开 · 加载 | `开关/01on/03加载` | 已开启，等待异步结果，滑块显示加载动效 |
| 关闭 · 常规 | `开关/02off/01常规` | 默认可交互，已关闭 |
| 关闭 · 禁用 | `开关/02off/02禁用` | 已关闭但不可操作，整体透明度 40% |
| 关闭 · 加载 | `开关/02off/03加载` | 已关闭，等待异步结果，滑块显示加载动效 |

> 禁用态通过对整个组件施加透明度实现，打开态禁用为 30%，关闭态禁用为 40%，两者不同。

## 尺寸规范

| 规格 | 宽 | 高 | 适用场景 |
|---|---|---|---|
| 常规（默认） | 52px | 32px | 单元格列表、常规场景 |
| 小号 | 39px | 24px | 特殊情况，空间受限 |
| 迷你 | 26px | 16px | 特殊情况，极小空间 |

> 宽高比约 1.625∶1，三档尺寸统一维持此比例。非必要不使用自定义尺寸。

## 典型使用场景

### 单元格列表内开关

配合 `单元格列表/02功能/01开关` 使用，Switch 右对齐放置于行尾，使用**常规尺寸（52 × 32px）**。

```
列表标题                          [Switch]
```

> 行高 52px，Switch 垂直居中，距右边距 16px（`padding-extra-loose`）。

---

## Constraints / Do & Don't

| | 规则 |
|---|---|
| ✅ | 单元格列表中开关使用常规尺寸（52 × 32px） |
| ✅ | 禁用态通过对整体施加透明度实现，不单独改变颜色 |
| ✅ | 危险操作在点击后加入二次确认流程 |
| ✅ | 自定义尺寸仅限特殊情况，推荐 24 或 16（高度） |
| ❌ | 不要在文案中加入"打开 / 关闭"等逻辑词 |
| ❌ | 不要在选项较多的场景中用 Switch 代替 Radio 或 Checkbox |
| ❌ | 不要将打开态禁用与关闭态禁用的透明度互换使用 |
